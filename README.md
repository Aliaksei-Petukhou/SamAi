# SamAI - UI Test Automation Framework

Этот проект представляет собой фреймворк для автоматизации UI-тестирования, созданный с использованием Python, `pytest` и `selenium`. Он был разработан совместно с CLI-агентом Gemini для демонстрации лучших практик в автоматизации тестирования.

## ✨ Особенности

- **Page Object Model (POM):** Код структурирован с использованием паттерна Page Object Model для улучшения читаемости, поддержки и переиспользования. Все Page Objects находятся в директории `pages/`.
- **Фреймворк Pytest:** Использует возможности `pytest` для обнаружения и запуска тестов, а также для гибких проверок (assertions).
- **Фикстуры:** Применяет фикстуры `pytest` (в `tests/conftest.py`) для управления общими ресурсами, такими как экземпляр WebDriver.
- **Продвинутые возможности `pytest`:** Демонстрирует использование параметризации (`@pytest.mark.parametrize`) и маркировки ожидаемо падающих тестов (`@pytest.mark.xfail`).
- **Интеграция с Git:** Включает файл `.gitignore` и следует стандартному рабочему процессу Git.

## 💻 Используемые технологии

- **Язык:** Python
- **Тестовый фреймворк:** `pytest`
- **Автоматизация браузера:** `selenium`
- **Система контроля версий:** `git`

## 🚀 Начало работы

### Требования

- Python 3.x
- `pip`
- `git`
- Браузер Google Chrome
- `chromedriver`, соответствующий вашей версии Chrome, должен быть доступен в системной переменной PATH.

### Установка

1.  **Клонируйте репозиторий:**
    ```shell
    git clone https://github.com/Aliaksei-Petukhou/SamAi.git
    cd SamAi
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```shell
    # Для Windows
    python -m venv .venv
    .venv\Scripts\activate

    # Для macOS/Linux
    python3 -m venv .venv
    source .venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```shell
    pip install -r requirements.txt
    ```

## ▶️ Запуск тестов

Чтобы запустить все тесты, выполните следующую команду из корневой директории проекта:

```shell
pytest
```

Вы увидите различные статусы выполнения тестов:
- `PASSED`: Тесты, которые успешно завершились.
- `xfailed`: Тесты, которые помечены как нестабильные из-за проблем на стороне сайта и от которых ожидается падение.
- `xpassed`: Тесты, которые были помечены как ожидаемо падающие, но неожиданно прошли, что подтверждает их нестабильность.
